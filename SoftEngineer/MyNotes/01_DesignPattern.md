# 设计模式浅析
需要工作经验的积累，抛砖引玉
## 目的(出发点)
设计模式和计算机领域中的大多数产物一样是用来解决问题的
解决问题
什么样的问题
变化(软件系统的发展伴随着变化的)
变化有什么问题
变化会引入复杂性，复杂性会引入bug和各式各样的麻烦
## 方法
用什么样的方法解决问题
抽象
抽象出可能的变化，并通过设计模式限制变化的方向
**refactor** to design pattern
